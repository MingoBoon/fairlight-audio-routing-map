The Build Spec

The complete, from-scratch specification of the 5.1 mixing template — tuned for a ~120-minute documentary-style piece delivered to broadcast, web and streaming. Build it once, save it as a .drp, reuse forever.

Verified against DaVinci Resolve Studio 21.0.2 · FlexBus · July 2026
1 · Mental model 2 · Bill of materials 3 · Build order (A–L) 4 · Delivery & loudness 5 · Quick reference 6 · New in Resolve 20/21
Version note. Modern Resolve (v17+) uses FlexBus: buses no longer have Main/Sub/Aux types — every bus is created identically (Fairlight ▸ Bus Format ▸ Add Bus: name, format, colour) and the Main/Sub/Aux labels in this spec describe roles that come from routing. Legacy fixed bussing still exists (Project Settings ▸ Fairlight ▸ "Use fixed bus mapping", new empty projects only, one-way conversion) but should not be used for new work.
Free vs Studio. Buses, VCAs, 5.1, FairlightFX and loudness metering all work in the free version. Studio adds immersive formats (5.1.2–9.1.6 / Atmos), Dolby Digital (AC-3) encoding, and the AI voice tools — so Studio is recommended for this delivery workflow.

1 · The mental model (read this first)

Signal flows in four stages. Nothing else in this document makes sense until this does:

SOURCE TRACKS  →  STEMS (sub buses)  →  PRINT MASTERS (main buses)  →  monitoring/delivery
                    ↑                        ↓
              reverbs (aux)            Webmix (compressed stereo)
  1. Source tracks hold your audio (dialogue, effects, ambience, music). They do not go straight to the speakers.
  2. Each track feeds one or more stems — a stem is a submix of one category (all dialogue, all effects, etc.). Stems are deliverables in their own right.
  3. Stems feed the Print Masters — the final mixes. There is a 5.1 Print Master and a Stereo Print Master.
  4. The Webmix is a compressed copy of the stereo master for quick web previews.

Two things ride alongside this flow: VCAs (control faders — no audio passes through; one fader rides a whole section) and reverbs (aux buses — tracks send to them; the reverb output returns into the matching stem).

M&E (Music & Effects) is the everything-but-dialogue deliverable used for foreign-language dubbing. It's fed from the FX and MX stems, so it stays automatically in sync. LFE (the ".1") is handled by dedicated LFE buses for effects and music, giving explicit control of what hits the subwoofer channel.

3 · Build order

Build buses before tracks where possible — Resolve lets you assign a track's output to a bus only once that bus exists. The order below avoids backtracking. (Prefer ticking things off? Use the interactive checklist.)

Step A — Set the project up

  1. File ▸ New Project → name it TEMPLATE — Fairlight 5.1.
  2. Project Settings (gear) ▸ Fairlight: set the timeline sample rate (48 kHz is standard for film/TV) and leave "Use fixed bus mapping" OFF so the project uses FlexBus.
  3. Move to the Fairlight page.
  4. Fairlight ▸ Bus Format… — this is where you declare buses. Open it before anything else.

Step B — Create all 21 buses (Fairlight ▸ Bus Format)

Add buses in this order and set each Format exactly per the table in §2: 3 mains (5.1 PM = 5.1; Stereo PM + Webmix = Stereo) → 4 × 5.1 stems → 4 stereo stems → 2 mono LFE buses → 8 reverb auxes (DX/FX: Mono, Stereo, 5.0 · MX: Stereo, 5.0). The Bus Format window only asks for name, format and colour — the Main/Sub/Aux roles come from routing (Step C).

Naming discipline matters: these names appear in every routing menu you'll touch. Keep them exactly as specified so the output-assign dropdowns stay scannable.

Step C — Bus-to-bus routing (Fairlight ▸ Bus Assign)

  • DX Stem → 5.1 Print Master
  • FX Stem → 5.1 Print Master and M&E Stem
  • MX Stem → 5.1 Print Master and M&E Stem
  • Stereo DX Stem → Stereo Print Master
  • Stereo FX Stem → Stereo Print Master and Stereo M&E Stem
  • Stereo MX Stem → Stereo Print Master and Stereo M&E Stem
  • FX LFE Bus → FX Stem (into the LFE channel) · Music LFE Bus → MX Stem (LFE channel)
  • Stereo Print Master → Webmix Stereo
  • Reverb auxes → their stems: DX Verbs → DX Stem · FX Verbs → FX Stem · MX Verbs → MX Stem
  • M&E Stem / Stereo M&E Stem: no onward routing — deliverables, not part of the final mix.
Why M&E is fed from stems, not tracks: it guarantees M&E always equals FX + Music + Ambience with zero per-track bookkeeping. Change an FX level and both the FX stem and M&E follow automatically.

Step D — Create the tracks

Add the 11 tracks with the correct widths per §2 (right-click track header ▸ Add Track ▸ format). Rename and colour-code each — colour coding is how you read the mixer at a glance. Resolve 21 tip: group them into Dialogue / FX / Ambience / Music track folders.

Step E — Assign track outputs to stems

Each track → its 5.1 stem and its stereo stem: Mixer ▸ Bus Outputs ▸ +, then + again (up to 10 outputs per track). Bulk: select several tracks, hold Alt/Option, click +. Matrix alternative: Fairlight ▸ Bus Assign.

Step F — Panning to the 5.1 field

  • Narration/VO → Centre (or phantom L/R if you prefer VO slightly wider than sync dialogue). Narration must always sit above the bed.
  • DX / Futz / PFX → hard Centre — dialogue lives in the centre channel.
  • FX A/B/C → spread across L/R/Ls/Rs; keep Centre mostly clear so effects don't fight dialogue.
  • Ambience A/B → wide, into the surrounds (Ls/Rs) for immersion.
  • Music 5.1 → passes through as discrete 5.1 · Music Stereo → L/R fronts.

Step G — LFE routing (the .1)

  1. On FX A/B/C, add a send to FX LFE Bus; insert a low-pass filter (~120 Hz) on the bus so only sub-bass passes.
  2. On Music tracks, add a send to Music LFE Bus; low-pass it the same way.
  3. Both LFE buses already route into their stems' LFE channel (Step C) — open each LFE bus's output panner and steer 100% to LFE.

Step H — Reverb sends (pre-routed)

  1. Insert FairlightFX Reverb on each aux: presets to suit width (room/plate for mono/stereo, hall/scoring for 5.0), 100% wet.
  2. Create track sends: Dialogue (DX, PFX) → DX Verbs — leave Narration/VO dry · FX → FX Verbs · Music → MX Verbs.
  3. Leave send levels at −∞ (pre-routed but silent) so the template opens clean. Post-fader is typical.

Step I — VCAs

  1. Fairlight ▸ VCA Assign: create the four VCA groups (Resolve supports up to 16).
  2. Assign member tracks per §2 (or per-track via the Mixer's Group dropdown).
  3. VCA masters appear right of the buses in the Mixer; they trim member faders and follow mute/solo — no audio passes through.

Step J — Monitoring + loudness metering

  1. Monitor the 5.1 Print Master (Control Room source in the monitoring panel).
  2. Configure 5.1 speakers in Preferences ▸ Video and Audio I/O.
  3. Set the stereo downmix type your broadcaster requires — most want Lt/Rt; some want Lo/Ro. Confirm per delivery spec.
  4. Loudness meters on BOTH print masters: pick the standard in the meter options (blue = in spec, yellow = tolerance, red = over); watch Integrated, Short-term, Momentary, True Peak and LRA; enable dialog-gated when checking the streaming target.

Step K — Webmix compression

On Webmix Stereo: Compressor (~3:1, moderate attack/release) then Limiter (ceiling −1.0 dBTP) — save both as one Chain FX preset. Aim ≈ −14 LUFS for web.

Step L — Save as template

Clean up (gains at unity, sends silent, mutes off) → File ▸ Export Project… for a reusable .drp. To reuse: import the .drp and start mixing.

4 · Delivery — loudness targets & export

The key fact: no single master satisfies broadcast + web + streaming. They use different loudness targets and different ways of measuring, so you mix once and produce a few normalized versions. Verify exact numbers against each recipient's current spec sheet.
DeliverableIntegrated loudnessTrue peakMeasurement / gating
US broadcast (ATSC A/85 · CALM Act)−24 LKFS−2 dBTPprogram (BS.1770)
EU broadcast (EBU R128)−23 LUFS−1 dBTPprogram-gated (+ control LRA)
Premium streaming (Netflix-style)−27 LKFS−2 dBTPdialog-gated
Web / social (YouTube, Spotify ~−14)−14 to −16 LUFS−1 dBTPprogram

Practical path: mix to a −24 LUFS / −2 dBTP program master (safe almost everywhere; watch LRA) → dialog-gated pass at −27 LKFS / −2 dBTP for streaming → Webmix lands at ~−14 to −16 LUFS / −1 dBTP → produce a −23 LUFS / −1 dBTP version for EU broadcast.

Export (Deliver page)

Render stems and masters as individual files: 48 kHz / 24-bit, channel order L R C LFE Ls Rs.

  • 5.1 Print Master → 6-channel WAV (or Dolby Digital/AC-3 via Studio) — the surround deliverable.
  • Stems (DX/FX/MX/M&E) → 6-channel WAVs — archival, QC, re-versioning. M&E is usually mandatory.
  • Stereo Print Master → stereo WAV (Lt/Rt or Lo/Ro per spec) · stereo stems + Stereo M&E → stereo WAVs.
  • Webmix → stereo AAC/MP4 or MP3 for quick online review.

Atmos: skip unless a platform requests it — 5.1 + stereo is a complete standard package for an information piece.

5 · Quick reference — who feeds what

  • Dialogue: Narration/VO (dry) + DX / Futz / PFX → DX Stem (+ Stereo DX Stem) → Print Masters. DX reverbs return to DX Stem. VO/DX drives dialog-gated loudness.
  • Effects: FX A/B/C + Ambience A/B → FX Stem (+ Stereo FX Stem) → Print Masters and M&E. FX LFE Bus → FX Stem LFE.
  • Music: Music Stereo/5.1 → MX Stem (+ Stereo MX Stem) → Print Masters and M&E. Music LFE Bus → MX Stem LFE.
  • M&E: FX Stem + MX Stem → M&E Stem (deliverable only). Stereo mirror likewise.
  • Masters: stems → Print Masters; Stereo PM → Webmix (compressed). Loudness meters on both print masters.
  • Control: four VCAs ride their sections — levels only, never routing.
  • Delivery in one line: −24/−2 (US) · −23/−1 (EU) · dialog-gated −27/−2 (streaming) · Webmix −14→−16. Always deliver M&E.

6 · New in Resolve 20 / 21 worth knowing

  • Audio track folders (v21): collapse the 11 source tracks into Dial/FX/Amb/Mus folders for a readable timeline.
  • Chain FX (v21): save the Webmix Compressor + Limiter as one reusable preset (up to 6 plugins).
  • 6-band Clip EQ (v21): per-clip EQ matching the track EQ — ideal for futz treatments; settings copy/paste between clip, track and plugin.
  • EQ Match / Level Matcher (v21): match a clip's level or tone to a reference — great for narration recorded across sessions.
  • AI Audio Assistant (v20, Studio): automatic first-pass mix to loudness targets — fine for temp mixes, not a replacement for this template's routing and deliverables.
  • AI Dialogue Matcher & Voice Isolation (v20, Studio): tone/level/room matching and noise removal for the DX tracks.
  • Loudness tools: colour-coded meters against your chosen standard, a live timeline loudness graph, right-click ▸ Analyze Audio Level, and a Cut-page mixer with metering (v20).
  • 21.0.1 / 21.0.2 are maintenance releases — no Fairlight routing changes.
